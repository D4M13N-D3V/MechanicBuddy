## Key Concepts

### Work Orders

A work order represents a single service job. It tracks:
- Which client and vehicle
- What work was requested
- Parts and labor used
- Final invoice

### Offers vs Repair Jobs

| Concept | Purpose |
|---------|---------|
| **Offer** | An estimate/quote for proposed work |
| **Repair Job** | Actual work being performed |

An offer becomes a repair job when the client accepts it.

### Client Types

| Type | Use For |
|------|---------|
| **Private Client** | Individual customers |
| **Legal Client** | Businesses, companies |

### Work Order Status

| Status | Meaning |
|--------|---------|
| **Created** | Work order just started |
| **In Progress** | Active work being done |
| **Completed** | Work finished |
| **Invoiced** | Invoice has been issued |

## Common Tasks Quick Reference

| Task | Navigation |
|------|------------|
| Create work order | Work Orders > + New |
| Find client | Clients > Search |
| Check vehicle history | Vehicles > Select vehicle |
| Create estimate | Work Order > Create Offer > Issue Estimate |
| Generate invoice | Work Order > Issue Invoice |
| Check inventory | Inventory > Search |
| Add new user | Settings > Users > + Add |
| Update company info | Settings > Company |
